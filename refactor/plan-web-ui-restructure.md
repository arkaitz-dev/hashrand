# Refactor Plan - Web UI Directory Restructure - 2025-08-09

## Initial State Analysis

### Current Architecture
- **Mixed File Structure**: JavaScript UI files mixed with Rust source code
- **Scattered Web Assets**: 
  - Components in `/src/components/`
  - CSS in `/static/css/`
  - Entry points in root (`/index.html`) 
  - Build config in root (`/vite.config.js`)
- **Legacy Files**: Old Web Components in `/static/js/` (unused)
- **Build Output**: `/dist/` directory (generated by Vite)

### Problem Areas
1. **No Clear Separation**: Frontend and backend code mixed in `/src/`
2. **Confusing Directory Names**: `static/` doesn't serve static files in dev mode
3. **Root Clutter**: Web config files in project root
4. **Legacy Code**: Old unused files still present

### Current File Mapping
```
Web UI Files (to be moved):
├── /index.html                    → /web-ui/index.html
├── /src/components/               → /web-ui/src/components/
│   ├── api-key-view.js
│   ├── generic-hash-view.js
│   ├── hash-generator.js

├── /package-lock.json            → /web-ui/package-lock.json
└── /vite.config.js               → /web-ui/vite.config.js

Legacy Files (to be removed):
├── /static/js/

│   ├── generic-hash.js (old, unused)

```

## Refactoring Goal

Create clean separation between Rust backend and JavaScript frontend by moving all web UI files to a dedicated `web-ui/` directory, while maintaining full functionality in both development and production modes.

## Workflow After Refactoring

### Development
```bash
# Terminal 1: Start web UI dev server
cd web-ui
npm run dev  # Runs on localhost:3000

# Terminal 2: Start Rust API server
cargo run -- --serve 8080
```

### Production
```bash
# Build web UI
cd web-ui
npm run build  # Outputs to ../dist

# Serve with Rust
cargo run -- --serve 8080  # Serves from dist/
```
